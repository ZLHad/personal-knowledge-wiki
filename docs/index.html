Core Operations

五大核心操作

覆盖知识管理全生命周期的自动化工作流

Ingest

知识摄入

对话、论文笔记一键摄入，LLM 自动提取实体、概念、规则，创建/更新 5-15 个 Wiki 页面，生成交叉引用。

深度 / 标准 / 浅度 三档

Query

知识查询

基于知识库语义搜索，回答带 [[wikilink]] 引用的结构化答案，可选归档为 Synthesis 综合页。

语义检索 + 综合归档

Lint

健康检查

8 条硬编码规则自动检测死链、孤儿页、标签一致性、源可追溯性等，附加软性质量分析。

8 Hard Rules + Soft Analysis

Migrate

知识迁移

批量扫描外部目录的 Markdown 文件，智能分类后自动创建 Raw 记录 + Wiki 页面，支持增量迁移。

目录 / 标准 / 深度 三模式

Export

知识导出

将 Wiki 导出为静态网站，支持 MkDocs Material、Quartz、纯 HTML 三种输出格式。

MkDocs / Quartz / HTML

从笔记到知识的四步流程

一次 Ingest 操作的完整旅程

01

投喂原料

将对话记录、论文笔记、文章摘录等原始材料保存到 raw/ 目录。每份原料写入即冻结，保证数据不可篡改。

raw/科研/论文资料/2026-04-08-paper-note.md 
# 论文阅读笔记
## 摘要
本文提出了一种基于深度强化学习的
动态资源调度方案...

## 核心方法
- 状态空间：信道状态 + 队列长度
- 动作空间：资源分配决策
- 奖励函数：加权时延-能耗...
02

LLM 编译

执行 ingest 操作，LLM 自动识别实体、提取概念、归纳规则，创建/更新多个 Wiki 页面，生成 [[wikilinks]] 交叉引用。

二分搜索 (Binary Search)

Definition

二分搜索是一种在有序数组中查找目标值的高效算法。每次比较将搜索范围减半，时间复杂度为 O(log n)。

Details

基本原理

  • 前提条件：数组必须有序
  • 核心操作：比较中间元素与目标值

变体对比

变体用途关键区别
标准二分精确查找找到即返回
左边界二分第一个 ≥ targetlo = mid + 1
右边界二分最后一个 ≤ targethi = mid - 1

Common Pitfalls

  • 忘记排序前提条件
  • 整数溢出：mid = lo + (hi - lo) / 2
  • 边界条件：lo <= hi vs lo < hi

HTTP 协议

Overview

HTTP (Hypertext Transfer Protocol) 是万维网数据通信的基础协议。定义了客户端与服务器之间请求-响应的标准格式。

Key Facts

  • 当前版本: HTTP/3 (基于 QUIC)
  • 默认端口: 80 (HTTP), 443 (HTTPS)
  • 特征: 无状态、基于请求-响应模型
  • 方法: GET, POST, PUT, DELETE, PATCH

Notes

  • HTTP/2 引入多路复用，解决队头阻塞
  • HTTP/3 使用 UDP 上的 QUIC 协议替代 TCP

Git 提交规范

Overview

规范化的 Git 提交信息能提高项目可维护性，方便自动生成 changelog 和追踪变更历史。

Rules & Patterns

  1. 格式: <type>: <description>
  2. 常用类型:
    • feat: 新功能
    • fix: 修复 Bug
    • docs: 文档更新
    • refactor: 重构
  3. 描述: 50 字符以内，动词开头

Examples

  • feat: add user authentication module
  • fix: resolve memory leak in parser
  • docs: update API documentation for v2
